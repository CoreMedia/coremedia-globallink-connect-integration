# GlobalLink Connect Cloud for CoreMedia Content Cloud v2406.3.0-1

Approve CMCC v12.2406.3.0.

## Summary

* Adapted and approved the connector for use with CoreMedia Content Cloud
  v12.2406.3.0.
* Updated `com.translations.globallink:gcc-restclient` to version 3.1.9.

## Upgrade Information 🆙

There is no need to upload the workflow definition again or to trigger
the `cm workflowconverter`.

Regarding the update of the GCC REST java client API, no adaptations are
required within the given code base. You may though, need to be aware of
the changes for your custom adaptations.

## Improvements 💪

### GCC REST Java Client Update

The GCC REST Java client has been updated to version 3.1.9. While no adaptations
are required within the given code base, the changes may be interesting for
your custom adaptations, like if you integrated context upload.

**Overview of Changes**

* **Context API Update:**

  * Context operation now uses `context-api/v4`.

  * There are now separate requests `UploadFileContextRequest` /
    `UploadImageContextRequest`. `ContextType` enum got removed.

  * `GCExchange.uploadContext(...)` returns `Context.ContextResponseData`
    (with `contextUrl` / `contextId`) rather than a plain static-URL String.

  * New methods:

    * `getContextConnectors`
    * `getContextPath`
    * `getContextConnectors`
    * `getContextHtml`
    * `getContextImage`

* `SubmissionSubmitRequest` includes an `owners_list` (owners) field.

* `TaskListRequest` includes the `node_ids` field.

* New Retry Configuration: Via `GCConfig` you can now configure retry logic for
  the REST client. Available configuration options are:

  * `maxRetriesOnServiceUnavailable`
  * `maxRetriesOnRequestErrors`

* New Initialization Opt-Out Configuration: `GCConfig` exposes boolean
  `validateOnInit` (default `true`). If set to `false`, initialization of
  `GCExchange` will skip the extra validation by calling `getConnectors()`.

* More robust HTTP connections: `HttpURLConnection` for accessing the REST API
  now default to timeout after 5 minutes for both connection and read timeouts.
